# PatchGuard Agent

The **PatchGuard Agent** is a lightweight, cross-platform Python daemon that runs on managed endpoints (Windows, Linux, macOS). It communicates with the PatchGuard backend via WebSockets to provide real-time telemetry, patch scanning, and automated installation.

## 🚀 Key Features

- **Real-Time Telemetry**: Reports CPU, RAM, and Disk usage every 60 seconds (configurable).
- **Automated Scanning**: Scans for missing OS updates using native package managers (`apt`, `yum`, `dnf`, `wusa`, `softwareupdate`).
- **Remote Execution**: Installs patches and triggers reboots on command from the central dashboard.
- **Resilient Connection**: Built with asynchronous WebSocket logic and exponential backoff reconnection.

## 🔑 How to get your API Key (`agent_api_key`)

The `agent_api_key` is a unique 32-character string generated by the PatchGuard backend for each device to authorize communication.

### 1. Register a New Device
Before deploying the agent, you must create a "Device" entry in the PatchGuard Dashboard:
1.  Go to **Device Inventory** in the PatchGuard UI.
2.  Click **Add Device**.
3.  Fill in the `hostname`, `ip_address`, and `os_family`.
4.  Once created, click on the **Device ID/Hostname** to open the **Device Detail** panel.
5.  Find the `agent_api_key` field and copy the value (e.g., `aZ12bY34cX56dW78eV90fU12gT34hS56`).

### 2. via Django Admin (For Administrators)
1.  Log in to `http://<your-server>:8000/admin/`.
2.  Go to `Inventory > Devices`.
3.  Select your device; the `Agent api key` will be listed in the detail view.

- **Python 3.12+**

- **Network Access**: Outbound connection to the PatchGuard FastAPI server (default port `8001`).

## 📦 Installation

### 🐧 Linux (Ubuntu, Debian) - `apt`
The easiest way to install on Linux is using the provided `install.sh` script.

```bash
# 1. Download and enter the agent directory
cd patchguard-agent/

# 2. Run the installer (requires sudo)
# Usage: sudo ./install.sh <AGENT_API_KEY> <SERVER_URL>
sudo ./install.sh "aZ12bY34cX56..." "ws://patchguard-server:8001/ws/agents"

# 3. Start the service
sudo systemctl start patchguard-agent

# 4. Verify (Checking for "Inst " upgrade preview logs)
sudo journalctl -u patchguard-agent -f
```

### 🦄 Linux (RHEL, CentOS, Rocky, Fedora) - `dnf` / `yum`
The `install.sh` script automatically detects your package manager.

```bash
# 1. Run the installer (requires sudo)
sudo ./install.sh "aZ12bY34cX56..." "ws://patchguard-server:8001/ws/agents"

# 2. Start and enable service
sudo systemctl enable --now patchguard-agent

# 3. Verify (Checking for "dnf check-update" logs)
sudo journalctl -u patchguard-agent -n 50
```

### 🪟 Windows (Server / Desktop) - `wusa.exe` / `PowerShell`
1. Install [Python 3.12+](https://www.python.org/downloads/windows/).
2. Open **PowerShell as Administrator** and navigate to the agent folder.
3. Setup and run:
   ```powershell
   # Setup Virtual Environment
   python -m venv venv
   .\venv\Scripts\activate
   
   # Install dependencies
   pip install -r requirements.txt
   
   # Run the Agent manually or via Task Scheduler
   python agent.py
   ```
4. **Patch Execution**: The agent uses `Get-HotFix` to scan and `wusa.exe /quiet /norestart` to install `.msu` packages provided by the backend.

### 🍎 macOS - `softwareupdate`
1. Ensure Python 3.12+ is installed.
2. Deployment steps:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip3 install -r requirements.txt
   python3 agent.py
   ```
3. **Patch Execution**: The agent uses the `softwareupdate` utility (`-l` for list, `-i` for install).

## ⚙️ Configuration (`config.yaml`)

The agent is configured via a YAML file. Defaults are provided, but you must set your specific `api_key` and `server_url`.

| Field | Default | Description |
|-------|---------|-------------|
| `server_url` | `ws://localhost:8001/ws/agents` | The FastAPI WebSocket endpoint. |
| `api_key` | `default_key` | Secret key used to authenticate with the backend. |
| `heartbeat_interval` | `60` | Frequency (seconds) for sending system metrics. |
| `log_level` | `info` | Logging verbosity (`debug`, `info`, `warning`, `error`). |
| `tags` | `[]` | List of custom tags for grouping in the dashboard. |

Example:
```yaml
server_url: "ws://192.168.1.10:8001/ws/agents"
api_key: "PG_AGENT_AX77_92"
heartbeat_interval: 30
tags:
  - "production"
  - "web-server"
```

## 🔍 Troubleshooting

### Logs
The agent logs to standard output. When running as a systemd service, use journalctl:
```bash
sudo journalctl -u patchguard-agent -f
```

### Common Issues
- **Connection Refused**: Ensure the PatchGuard backend (`realtime` service) is running and accessible over the network.
- **Permission Denied**: The agent **must** run as root/Administrator to list or install updates.
- **Invalid API Key**: Check the `X-Agent-Key` log messages; if the backend rejects the key, the agent will disconnect immediately.
